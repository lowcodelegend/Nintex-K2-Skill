# Design and compatibility boundary

## Modern authoring path

The installed HTML5 application lives under `K2 smartforms Designer\K2 workflow Designer`. Its client calls `api/workflow/loadAndLockWorkflow`, `api/workflow/savejson`, and `api/workflow/unlockworkflow`. Save/publish submits the JSON process, process/category identifiers, client lock identifier, and a deploy flag.

`k2wf` mirrors that path through `K2DesignerManagementClient.SaveKprx`. The modern server pipeline deserializes `K2Process`, builds deployment XML, and deploys it when requested. Published runtime verification and deletion use `WorkflowManagementServer`; this does not make the legacy runtime format the authoring contract.

Some concrete on-prem provider namespaces contain the word `Legacy`. They are adapters behind this installed modern web application. That namespace does not authorize use of the older `SourceCode.Workflow.Design` or `SourceCode.Workflow.Authoring` design-time object models.

## Observed K2 Five 5.10 contract

- Designer environment token: `smartforms`.
- Current workflow designer schema: `14`.
- JSON root component: `50001`.
- A minimal deployable definition needs Start/End activities, connector/port references, and the process/activity configuration components generated by the designer.
- Publishing returns separate saved JSON and deployed runtime identifiers/version information.
- `SaveKprx(close=true)` does not reliably clear the HTML5 designer lock. Invoke `UnlockProcess` with the identity resolved by `ConnectionClassContext.GetUser`. Do not call `GetUserProcessKprx` or `GetProcessJson` afterward: on this 5.10 build those reads check the workflow out again and can create the two-attempt unlock loop. Read metadata with `GetProcessInfo`, then fetch the exact JSON through `GetProcessDefinitionPerVersion`.
- K2 prevents direct deletion of the default runtime definition. Safe cleanup verifies zero instances, unsets the default, deletes definitions, then removes designer JSON.
- Activity and connector `systemName` values are container keys, not merely labels. Reusing `DefaultLine` fails compilation when a graph has multiple connectors; follow K2's `DefaultLine`, `DefaultLine 1`, ... convention. Parent activities keep type names such as `SmartObject Method`, `Send Email`, and `Task`, while child events carry friendly titles.
- SmartObject method events use external-reference, method, and input metadata plus property-mapper token references. Resolve input ordering/types with `SmartObjectClientServer`; do not infer them from SQL column order. Set friendly SmartObject/method titles and the `radNoOutputs` mode so the configuration panel can reconstruct the selected method and mappings.
- SmartObject events also need the hydrated smart-wizard definition and `SmartObject_Service_Functions` reference for later browser configuration. Compilation alone succeeds without them, but the Designer config panel expects `event.wizardDefinition` when its dynamic wizard cache is empty. Every `GetSmartObject*`/`GetDefaultLoadMethod` mapping inside that embedded definition must point to the actual service-functions external-reference ID; pointing them at the request SmartObject leaves Input Mappings visibly empty even though saved `pmInputs` values exist.
- K2's built-in User Task notification is `sendNotification=true` plus a component-30006 `emailConfiguration`. Customized subject/body fields are ordered smart-field arrays; primary item-reference properties use component 10011, participant/worklist fields use component 10009, and the worklist hyperlink uses the HTML helper function expression.
- Direct User Task destinations mirror the manifest in order. `$originator` becomes the dynamic `ProcessOriginatorFQN` User expression; explicit K2 identity strings become literal User destinations. Verification compares every emitted destination with the manifest.
- Task action references and parent outcomes must target the same action internal IDs. Pass the standard `SN` serial parameter to the task form and add the request identifier separately when required.
- Connector `ui.path` values are absolute canvas coordinates, not reusable relative shapes. Derive every path from its source/destination activity coordinates or Designer initially hides the line until it recomputes the route.
- Native integration is a cross-artifact contract. A Start form rule executes `StartProcess` and maps the primary Create reference; a Task state loads by `SerialNo` and submits the selected action. Use `SmartFormsManagementProvider.UpdateForm`/`PublishClientEvent`, the same providers called by the HTML5 wizard, rather than replacing form XML.

## Boundary

Keep `start-end` immutable as the smoke-test template. `request-approval` owns its tested topology and does not accept arbitrary component IDs or opaque fragments. Raw `json-file` remains the schema-validated expert escape hatch.
